13.0 · principio

Principio

Una guía sin archivos no es una guía. Es un folleto. ForMeta entrega todo lo que define.

Cada capítulo del manual produce assets descargables: tokens CSS, SVGs canónicos, helpers JS, plantillas listas. Esta sección es el índice maestro — un solo sitio donde encontrar todo lo que ha generado el sistema. Cuando alguien empiece una pieza de marca nueva, parte de aquí.

13.6 · glosario

Glosario

Términos del manual definidos. Si una palabra aparece varias veces y tiene un significado específico, vive aquí.

ForMeta
CAP. 02

Nombre del estudio. Nace de formita, palabra mallorquina para dar forma a las cosas. Dos personas en Mallorca construyendo IApps para empresas reales.

Formitar
CAP. 02

Verbo mallorquín. Dar forma. La marca lo aplica al software: coger el conocimiento que ya existe en una empresa y darle forma digital.

IApp
CAP. 02

Intelligent Application. Aplicación donde la IA es la capa lógica central, no una función añadida. Entiende contexto, aprende del uso, decide. El trabajador deja de ser intermediario entre datos y acción.

Halftone
CAP. 03 · 06 · 07

Técnica de representación por densidad de puntos. En ForMeta es la gramática visual única: el símbolo, los marcadores editoriales y las composiciones comparten la misma lógica a escalas distintas.

Núcleo
CAP. 03

El punto terra sólido en el centro del símbolo. La parte sólida única — todo lo demás son dots ink. Representa la inteligencia contenida.

Gota
CAP. 03

La silueta del símbolo. En el sistema halftone es implícita: no hay trazo, los puntos definen el borde por densidad.

Capa 01–04
CAP. 05

Las cuatro capas del sistema cromático: 01 sólida (paleta base), 02 gradientes editoriales, 03 cristal líquido, 04 espectro generativo. Cuanto más alta la capa, más restringido su uso.

Liquid glass
CAP. 05 · 10

Capa 03 del color. Superficie translúcida con backdrop-filter: blur(). Hereda color del fondo, no tiene tono propio. Cards, modales, dropdowns.

Expressive-out
CAP. 08

Curva de easing canónica: cubic-bezier(0.16, 1, 0.3, 1). Acelera fuerte al inicio y se asienta despacio al final. La firma de motion editorial.

Metadata strip
CAP. 04

Tira de metadatos en Plex Mono al pie de cada pieza. Versión, fecha, localización, ID. Firma técnica como ornamento.

Halftone editorial
CAP. 06

Familia de marcadores en capa 05 del icon system: burst, strip, bullet, grain, corner, focal. Para piezas editoriales largas — leadlines, dividers, watermarks.

Container narrow/default/wide
CAP. 09

Tres anchos canónicos: narrow 720px (long-form), default 1200px (layouts), wide 1440px (dashboard). Elegido por función, no por estética.

13.7 · stack

Stack técnico

Herramientas que el estudio usa para construir piezas con el sistema. No es prescriptivo — cualquier herramienta que respete los tokens vale. Esta es la lista de las que se han usado para este manual.

HTML + CSS plano
Manual · landings

El manual está escrito en HTML + CSS sin build. Importable directamente. Sin compilación, sin tooling.

CSS Custom Properties
Tokens · theming

Toda la sistema vive como variables CSS. Cambiar un token actualiza todo el sistema. Nada hardcoded.

SVG inline
Símbolo · iconos · charts

Todo lo gráfico es SVG. currentColor compatible. Sin dependencias de imagen raster.

JS vanilla
Halftone generator · componentes

Sin frameworks. Canvas.isPointInPath para halftone, backdrop-filter para liquid glass, listeners para componentes.

Claude Code
Construcción del manual

El manual se construyó iterativamente con asistencia AI. Cada capítulo revisado y validado en navegador antes de pasar al siguiente.

JSZip
Packs descargables

Library MIT para empaquetar assets en zips desde el browser. Único dependency external.

13.9 · gobernanza

Responsables

La guía la aprueban los dos fundadores. Cambios fundacionales (manifiesto, logo, color base, tipografía) requieren acuerdo explícito de ambos y bump de versión. Cambios en aplicación (componentes, plantillas, documentación) se hacen por consenso informal y se documentan en el siguiente ciclo.

Toda nueva pieza pasa por la pregunta del capítulo 01: ¿esto es coherente con el manifiesto de la sección 02? Si la respuesta es no, no se hace. Si la respuesta es sí, se hace y se documenta aquí para la siguiente versión.

Volúmenes 3D halftone

El lenguaje halftone proyectado en 3D. Cada figura es un point cloud distribuido sobre una superficie matemática — esfera (Fibonacci spiral), gota revolucionada (silueta 2D extruida 360°), torus (donut). Proyección ortográfica, tamaño y opacidad por profundidad Z, núcleo terra al frente.

3D.A · esfera
Esfera halftone

600 puntos sobre superficie esférica (Fibonacci spiral). Frontales grandes y opacos, traseros pequeños y tenues. Tilt 20° para perspectiva.

600 pts · R 100 · proyección ortográfica
3D.B · gota
Gota volumétrica

La silueta 2D del símbolo (cap. 03) revolucionada 360° en torno al eje vertical. Dots distribuidos en la superficie. El núcleo terra queda al frente.

500 pts · revolución 360° · núcleo z=máx
3D.C · torus
Torus halftone

Donut paramétrico — radio mayor R=80, radio menor r=30. 700 puntos distribuidos en la superficie con tilt 25° para vista superior. Para piezas técnicas y poster.

700 pts · R 80 · r 30 · tilt 25°

Aplicaciones: los volúmenes 3D se usan en piezas editoriales largas (artículos, presentaciones), heros de portada, ilustración de conceptos (datos, inteligencia, sistema). Restricción: nunca en UI funcional ni componentes — pertenecen al registro expresivo.
